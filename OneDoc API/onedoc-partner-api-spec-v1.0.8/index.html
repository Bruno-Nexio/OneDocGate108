Partner API Spec - OneDoc sync v1.0.8

Scroll down for code samples, example requests and responses. Select a language for code samples from the tabs above or the mobile navigation menu.

This document specifies a few API endpoints that can be provided by Partner Software to enable event synchronization with the OneDoc online booking platform.

Formats

All dates and datetimes are formatted according to the ISO8601 standard:

  • A date is formatted as yyyy-MM-dd, e.g. 2018-03-19
  • A date with local time is formatted as yyyy-MM-dd[T]HH:mm:ss, e.g. 2018-03-19T10:30:00
  • A timestamp is formatted as yyyy-MM-dd[T]HH:mm:ss.SSS[Z], e.g. 2018-03-19T15:36:27.817Z where Z denotes UTC time

Authentication

  • HTTP Authentication, scheme: basic

Events

OneDoc will proceed as follow to synchronize events.

From OneDoc to Partner Software

  • Event created on OneDoc => CreateEvent
  • Event updated on OneDoc => UpdateEvent (optional)
  • Event cancelled on OneDoc => DeleteEvent

From Partner Software to OneDoc

Initial import phase
This phase runs only once for every new syn activation:

  • Initially retrieve the latest event change id, using GetLatestEventChangeId which will serve as the starting point for change handling
  • Initially import all future events, month by month, using GetEvents for a given time window (e.g. 2 years)

Change handling
This phase runs continuously until the sync is deactivated:

  • Every ~30 seconds, retrieve changes and apply them to OneDoc using GetEventChanges and GetEvent
  • Every day, using GetEvents, fetch the next last day within the limit time window (e.g. today + 2 years) in order to import future recurrence instances to OneDoc

GetEvents

Code samples

GET /calendars/{calendarId}/events?fromDateTime=2020-04-01T00%3A00%3A00&untilDateTime=2020-05-01T00%3A00%3A00 HTTP/1.1

Accept: application/json

GET /calendars/{calendarId}/events

Return the list of events occurring within the provided datetime interval. Deleted events must not be returned.

The filtering condition should be startDateTime >= fromDateTime && startDateTime < untilDateTime.

Pagination should not be needed as OneDoc will never query more than 1 month of events at a given time.

This endpoint must expand recurring events and return all their instances within the provided datetime interval

Parameters

Name In Type Required Description
calendarId path string true ID of the calendar from which events should be retrieved
fromDateTime query string true Inclusive startDateTime lower bound
untilDateTime query string true Exclusive startDateTime upper bound

GetEventChanges

Code samples

GET /events/changes?afterChangeId=9974231 HTTP/1.1

Accept: application/json

GET /events/changes

Return the IDs of all events whose changeId > afterChangeId.

A change should be generated when a Event is changed, specifically in the following cases:

  • It is newly created
  • It is updated
  • It is cancelled/deleted
  • Its recurrence rule is modified or one of its instance is overridden
  • It is moved to another calendar

The changes must be returned in an ordered fashion: globally sorted by changeId in an ascending manner (earliest changes first). The ordering requirement also means that there should exist a changeId value that is smaller than all the others (minimum).

There should be a limit of changes returned at once, e.g. 100.

This endpoint would ideally return all event changes, i.e. accross all synchronized calendars. This would allow OneDoc to perform a single request to retrieve all event changes (changeId must be global). If this is not feasible, the calendarId parameter can be added to the endpoint's path but one request per calendar will have to be performed.

Parameters

Name In Type Required Description
afterChangeId query integer true Exclusive lower bound for changeId

This endpoint is used only once during the initialization of a new synchronization to retrieve the current state. After the initial import of events, the change handling mechanism will continue from this latest change id instead of processing all changes from the very beginning.
